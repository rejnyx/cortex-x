## Risks

- **TruffleHog binary not in test environment** — the secret-sweep test must spoof PATH or override the binary-discovery env var. If the test relies on real TruffleHog and the CI runner doesn't install it, the test will be flaky.
- **qlty schema drift** — the qlty CLI's JSON output format is not formally specified. The mock must mirror a recent real-run snapshot; if qlty bumps its schema, the test goes stale silently. Mitigation: include a comment in the test pointing at the snapshot's date + a refresh procedure.
- **Senior-tester 39-smell registry is internal data** — the test needs fixture test files that trigger specific smell IDs. The registry lives in `bin/steward/actions/senior-tester/_lib/smell-registry.cjs` (or similar) — the test must mirror its structure, not reimplement. Coupling is intentional; refactors to the registry will require test update.
- **Phase B opt-in path (LLM judge) has historically silently misfired** — Sprint 2.11 dogfood revealed cases where the gate var was checked too late. The test must assert the gate fires BEFORE any fetch attempt; use a fetch mock that records all invocations and assert count==0 when env var is unset.
- **Key-probe is the only cron in this list that talks to a real external endpoint by design** — the test must replace the HTTP boundary with a mock, but the mock interface needs to match real OpenRouter's `/auth/key` response shape (success has `data.label`, failure has `error.type`). Wrong shape = test passes but real cron breaks.
- **Cron-workflows-coverage extension creates a self-referential check** — the new inventory test asserts "the 5 test files exist". If the test files exist but are themselves broken (e.g., a syntax error), the inventory passes while the underlying tests don't run. Mitigation: AC-2 (`node --test` exits 0 on all 5) catches this; both tests are required.
