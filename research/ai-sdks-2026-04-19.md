# Research: AI Agent SDKs (2026-04-19)

Input to [`standards/ai-sdks.md`](../standards/ai-sdks.md). Refresh every 6 months or on SDK major bump.

## 1. Claude Agent SDK (Anthropic)

**Package:** `@anthropic-ai/claude-agent-sdk` (TypeScript) · `claude-agent-sdk` (Python)

**Renamed from:** Claude Code SDK → Claude Agent SDK (late 2025). Reflects broader positioning: not just coding agents.

**What's in it (April 2026):**
- `query()` async-iterator API returning a stream of messages
- Built-in tools: Read, Edit, Write, Bash, Grep, Glob, WebFetch, WebSearch, Task (subagents), TodoWrite, NotebookEdit
- **Skills** — progressive disclosure of large instruction bundles (shipped Oct 2025). Skills are markdown + assets loaded on demand via the Skill tool. Reduces context bloat.
- **Hooks** — PreToolUse / PostToolUse / SessionStart / PreCompact lifecycle hooks executed by the harness
- **Subagents** — isolated context windows, delegated via the Task / Agent tool
- **MCP support** — first-class, both as server (expose tools) and client (consume tools)
- **Session management** — forking, `get_context_usage()`, automatic compaction
- TS package bundles the Claude Code binary; Python package spawns it

**Language parity:** TS and Python are first-class. API shapes differ slightly but feature set is the same.

**Distribution:** Free, open-source. Pay only API tokens.

**Use cases in the wild:**
- Headless Claude Code for CI (code review agents, automated refactor bots)
- CLI tools that delegate to Claude for generation + editing
- Long-running worker agents (research, doc generation, data pipeline orchestration)
- Self-hosted coding assistants

**Weaknesses:**
- Claude-only — no Gemini/GPT
- Heavy runtime — not suitable for edge / serverless cold-start
- Overkill for simple chat UIs (use Vercel AI SDK there)

**Sources:**
- https://docs.claude.com/en/api/agent-sdk/overview
- https://github.com/anthropics/claude-agent-sdk-typescript
- https://github.com/anthropics/claude-agent-sdk-python

## 2. OpenAI Agents SDK + Codex SDK

**Packages:** `openai-agents` (Python), `@openai/agents` (TypeScript), `@openai/codex-sdk`

**What's in it (April 2026):**
- April 2026 harness update: configurable memory, sandbox-aware orchestration, `apply_patch`, Skills (progressive disclosure), AGENTS.md instructions, subagents, code-mode
- Multi-agent orchestration: agents with `handoffs`, `Runner.run()` / `Runner.run_streamed()`
- Works with **100+ models** via Chat Completions adapter — NOT OpenAI-only despite the name
- Guardrails, structured output, tracing built-in
- Codex SDK wraps the Codex CLI / agent for programmatic autonomous developer workflows

**Language parity:** Python leads. TypeScript shipped but harness/sandbox TS support rolling out through 2026.

**Sandboxing:** April 2026 update introduced a configurable sandbox (Docker/Firecracker-style) for safe code execution. Significant differentiator vs Claude Agent SDK (which relies on OS-level file permissions + hooks).

**Use cases:**
- Python-native backends (FastAPI, Django, data pipelines)
- Sandboxed codegen (PR bots, auto-migration tools)
- Multi-agent systems (routing, handoffs, specialist agents)
- Enterprises standardized on OpenAI

**Weaknesses:**
- TS lags Python — check feature parity before shipping in TS
- Sandboxing infra = operational complexity
- Responses API has documented `tool_call_id` bugs with gpt-5.x — use Chat Completions for function calling

**Sources:**
- https://openai.com/index/the-next-evolution-of-the-agents-sdk/
- https://developers.openai.com/codex/sdk
- https://openai.github.io/openai-agents-python/

## 3. Vercel AI SDK

**Package:** `ai@6` + provider plugins (`@ai-sdk/anthropic`, `@ai-sdk/openai`, `@ai-sdk/google`, …)

**What's in it (AI SDK 6, Q1 2026):**
- Unified `generateText` / `streamText` / `generateObject` / `streamObject` across providers
- Multi-step tool loops with `stopWhen: stepCountIs(N)` (or custom predicates)
- `ToolLoopAgent` class for declarative agent definition
- Human-in-the-loop approval flow for tool calls
- AI SDK DevTools — debug tool calls, messages, cache hits in browser
- `experimental_telemetry` → OpenTelemetry spans
- AI Elements — React streaming UI primitives (message bubbles, reasoning, tool-call UI)
- `experimental_mcpClient` — consume MCP tools

**Language support:** TypeScript only. Frameworks: Next.js (primary), React, Vue, Svelte, Solid, Node.

**Use cases:**
- Streaming chat UI in web apps
- Provider-agnostic AI features (swap models with one import)
- Tool-calling agents embedded in web requests
- RAG pipelines with streaming generation
- Any `/api/chat` endpoint

**Strengths:**
- Best streaming UI story in the ecosystem
- Model-agnostic by design — avoid vendor lock-in
- Small, focused API surface
- Excellent Next.js + React integration

**Weaknesses:**
- TS only — Python backends can't share this layer
- No filesystem/shell built-in tools (BYO)
- Agent loop simpler than Claude Agent SDK / OpenAI Agents SDK
- Not designed for long-horizon autonomous work (minutes-to-hours loops)

**Sources:**
- https://vercel.com/blog/ai-sdk-6
- https://ai-sdk.dev/docs/introduction
- https://github.com/vercel/ai

## Head-to-head

| Dimension | Vercel AI SDK | Claude Agent SDK | OpenAI Agents SDK |
|---|---|---|---|
| Primary language | TS | TS + Python | Python (TS catching up) |
| Model lock-in | None (provider-agnostic) | Claude-only | None (100+ via Chat Completions) |
| Streaming UI | Best-in-class | N/A (headless) | Basic |
| Filesystem / shell | BYO | Built-in (10+ tools) | Via sandbox |
| Skills / hooks | ❌ | ✅ | ✅ (Apr 2026) |
| Subagents | Via orchestration | Built-in | Built-in (handoffs) |
| MCP | Client (experimental) | Native server + client | Via bridge |
| Sandboxing | N/A (runs in web process) | OS perms + hooks | Docker/Firecracker harness |
| Long-horizon autonomy | Weak | Strong | Strong |
| Cold-start / edge | ✅ | ❌ (heavy) | ❌ (heavy) |
| Cost | Tokens only | Tokens only | Tokens + sandbox infra |

## Meta-recommendation for cortex-x

1. **Vercel AI SDK = default for web tier.** All `nextjs-saas`, `chatbot-platform`, `waas-template` scaffolds. Matches TS/Next.js target, owns streaming UI, no vendor lock-in.

2. **Claude Agent SDK = default for autonomy tier.** `ai-agent` and `cli-tool` (when AI-primary). Deep Claude Code alignment (cortex-x itself targets Claude Code).

3. **OpenAI Agents SDK = opt-in.** For Python-heavy backends or sandboxed codegen needs.

4. **Two-SDK topology is fine** and common: Vercel AI SDK for the web chat surface + Claude Agent SDK (or OpenAI Agents SDK) for the background autonomous worker. Profiles model this via `ai_sdk` + `ai_sdk_web`.

5. **Do NOT build a unified abstraction** across all three. Vercel AI SDK already abstracts providers. Execution models of Claude Agent / OpenAI Agents differ too much (Skills, hooks, sandbox semantics) — a wrapper would be leaky. Treat them as distinct "autonomy tier" choices.
