# Research: AMD Developer Hackathon 2026

## Key insights (TL;DR)

1. **MI300X 192GB = the story.** Pitch must feature "one GPU can't do on H100" moment. Llama-3.3-70B FP16 + 128K context on single card = hero demo.
2. **Avoid beginner track saturation.** Hack_AI / CashCat / PowerPibes / BabaTilluGang already own generic/fintech/business-case slots.
3. **Winning formula:** (saturated agent archetype) × (unusual constraint/domain/UX) + (visible multi-agent handoff) + (production polish).
4. **Enterprise judge panel** (Netflix, Apple, JP Morgan, PayPal, AMD, Amazon, Hippocratic) → B2B ROI framing with quantified metrics beats consumer delight.
5. **Dave's asymmetric advantage:** 12-day RELO agent fork (safe-tool + multi-step + 3-layer memory + background tasks) = instant depth vs chatbot-wrapper hackathon entries.

## AMD stack (April 2026)

**Hero model:** `meta-llama/Llama-3.3-70B-Instruct` FP16 on single MI300X TP=1, 128K ctx.
**Reasoning big-gun:** `DeepSeek-V3.1` on 8×MI300X via AITER MLA (21K tok/s decode reported by Moreh).
**Multimodal fallback:** `Qwen-2.5-VL-72B`.

**Serving:** `rocm/vllm` Docker, v0.8+. Env:
```
VLLM_ROCM_USE_AITER=1
VLLM_ROCM_SHUFFLE_KV_CACHE_LAYOUT=1
VLLM_USE_TRITON_FLASH_ATTN=0
PYTORCH_TUNABLEOP_ENABLED=1
```

**Budget:** $100 = ~50h single MI300X ($1.99/hr) OR ~6h 8×MI300X ($15.92/hr). Build on 1×, demo on 8×.

**Gotchas:**
- Ubuntu 24.04 + Ray = HIP invalid-device-ordinal → pin 22.04 or skip Ray
- Sliding-window models (Gemma, Mistral-Small) slow → avoid
- Model download (140GB FP16) ~15 min cold → cache early
